# Amplify deployment from GitHub | Automatic deployment
A responsive glasses store website showcasing different types of eyewear. Built with HTML and CSS and deployed using GitHub and AWS Amplify for cloud learning purposes.

-AWS console 
-AWS Amplify 
-Route 53

## Steps to Deploy
Steps to Deploy
1.	Created a GitHub repository for the website and uploaded all project files and images. 
2.	Deployed the application using the GitHub repository in AWS Amplify. 
3.	Granted AWS Amplify permission to access only the required repository (e.g., glasses-store-website). 
4.	Saved all settings and clicked Deploy App. 
5.	AWS Amplify generated a default domain (for example: https://main.d1fkbnkl6p7ypg.amplifyapp.com/). 
6.	Opened the generated domain in a new browser window and verified that the website was running successfully. 
Automated Build and Deployment
7.	Made changes to the HTML file in GitHub and committed the updates. 
8.	AWS Amplify automatically detected the changes and started a new deployment. After successful deployment, the updates appeared on the website. 
Custom Domain Configuration
9.	Navigated to Custom Domain in the left panel of AWS Amplify. 
10.	Created a custom domain, selected Amplify-managed certificate, and completed the website hosting with the custom domain.
